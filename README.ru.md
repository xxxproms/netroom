# NetRoom

Документация сети для IT-отдела — филиалы, серверные, шкафы, свичи, патч-панели, порты, VLAN, кабельные трассы и IP-адреса в одном месте вместо таблицы, которой никто не доверяет.

> **Статус:** ранняя разработка. Каркас (аутентификация, роли, локализация) готов; предметные разделы выходят фаза за фазой.

## Зачем

Обычно сеть ведут в Excel: лист на каждый свич, членство во VLAN закодировано заливкой ячеек, и на вопрос «какой порт обслуживает это рабочее место» ответить нельзя, не спросив того, кто это строил. NetRoom сохраняет привычную модель — матрицу «VLAN × порт», — но делает её тем, что можно искать, трассировать и показывать коллегам.

## Возможности

- **Филиалы и помещения** — несколько площадок, у каждой свои серверные, шкафы и рабочие места.
- **Шкафы** — схема юнитов, устройства с привязкой к позиции, фронт и тыл.
- **Свичи** — каталог моделей с шаблонами портов: создаёшь устройство — порты создаются сами.
- **VLAN-матрица** — та самая сетка «VLAN × порт» с tagged/untagged и проверкой PVID.
- **Патч-панели** — пары фронт/тыл, поэтому трасса кабеля проходит *сквозь* панель до розетки.
- **Кабельный журнал** — медь и оптика (1 или 2 волокна), метки, длины и сквозная трассировка от порта свича до рабочего места.
- **VPN-туннели** — как филиалы связаны между собой, каким бы файерволом ни терминировались.
- **IPAM** — подсети, занятость адресов, конфликты, management-адреса рядом со своими устройствами.
- **Карта сети** — интерактивная схема в двух уровнях: все филиалы и туннели между ними, затем устройства внутри филиала.
- **Роли и области видимости** — права определяют, *что* можно делать, доступ к филиалам — *где*.
- **Журнал изменений** — каждая правка со значениями до и после.

## Требования

- PHP 8.3+
- Node 22+
- PostgreSQL 16+ (для быстрой пробы подойдёт SQLite)

## Быстрый старт

```bash
git clone https://github.com/xxxproms/netroom.git
cd netroom
composer setup
composer dev
```

`composer setup` установит зависимости, создаст `.env`, сгенерирует ключ приложения, выполнит миграции и соберёт фронтенд. `composer dev` поднимет приложение, обработчик очереди, просмотр логов и Vite одной командой.

Учётные записи создаёт администратор — самостоятельная регистрация выключена. Чтобы открыть её, поставьте `NETROOM_ALLOW_REGISTRATION=true` в `.env`.

### Демо-данные

Чтобы посмотреть панель с данными, а не пустой:

```bash
php artisan db:seed --class=DemoSeeder
```

Создаётся вымышленная сеть — два соседних комплекса с общим VLAN-планом, городской офис, фабрика и коттедж — со шкафами, свичами, патч-панелями и описанными портами. Заодно заводятся четыре учётные записи, по одной на роль (`admin@`, `engineer@`, `tech@`, `viewer@example.com`, пароль `password`), чтобы сравнить уровни доступа. Только для тестовой базы, не для рабочей.

### Docker

```bash
cp .env.example .env
docker compose up -d
```

Приложение поднимется на <http://localhost:8080> вместе с PostgreSQL.

## Язык

Интерфейс есть на русском и английском. `APP_LOCALE` задаёт язык по умолчанию, каждый пользователь может выбрать свой в разделе **Настройки → Оформление**.

## Разработка

```bash
composer ci:check   # линтеры, формат, типы и тесты — всё, что гоняет CI
composer test       # только тесты
npm run lint        # автоисправление фронтенд-линтера
```
